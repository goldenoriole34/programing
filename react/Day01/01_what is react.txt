리액트는 프레임워크인가 라이브러리인가?
  구글 검색하면 웹 프레임워크라고 함
  공식 홈페이지에는 라이브러리라고 함

  개인적으로 프레임워크라고 생각하는 경우도 있음


프레임워크 라이브러리 차이점
  ㄴ라이브러리는 폴더구조가 없음
  ㄴ프레임워크는 폴더구조가 있음


따라서 리액트는 크게 4가지로 나뉘고 있고 프레임워크라고 보는 것이 더 적합하다.


리액트는 상태가 변경되는 값만 파악하여 재랜더링 시키는 것이 특징이다.
js로는 div 항목 전체가 바뀌지만 react는 항목 내 변경사항만 변경된다


리액트는 최소 2개의 라이브러리만 있으면, 리액트를 만들 수 있다.
  1) react : react문법이 담겨져 있는 라이브러리
    3) babel : jsx => js영역에서 HTML을 사용할 수 있게 해줌 - 페이스북이 차용해옴
  2) react-dom : 변경되는 값을 체크하여 해당 영역만 변경시키는 역할
  4) webpack - 페이스북이 차용해옴

리액트는 SPA를 만들기 위해 사용함
ㄴ 싱글페이지 어플리케이션


리액트로 버튼을 만드는 방법(원리)
  React-dom.render()
  첫번째 인자값 : 내가 넣을 내용 
  두번재 인자값 : 내가 넣을 위치 => document.querySelector('#root')
  
  인자 값이 3개 들어감
    (1) 엘리먼트 명 => a, button, input, li 등등
    (2) 속성값(object) => <input type="text">
    (3) innerHTML 내용물 => <span>하이</span>


컴포넌트는 Element를 함수나, 클래스에 담는 행위로 재활용이 가능한 단위이다.
종류는 2가지가 있다
  ㄴ클래스형 컴포넌트(과거에 많이 사용)
  ㄴ함수형 컴포넌트(현재 주로 사용)


컴포넌트는 JSX형식으로 사용함

ReactDOM은 기본적으로 하나의 컴포넌트만 실행해줌

webpack이란?
  ㄴ 브라우저와 통신하는 여러개의 복잡한 js파일을 단 하나의 js파일처럼 번역해주는 번들
  ㄴ 싱글 페이지 어플리케이션을 여러인원이 개발하며 변수명 등이 충돌하는 것을 방지
  ㄴ 웹팩 대용으로 Vite가 나왔다


브라우저 - (요청/반응) -> 프론트서버 -> 백엔드 서버 - (ㄴComponentDidMount 비동기통신) -> 브라우저
